Technical Deep Dive

How Retention Optimizer works

A churn prediction engine that combines LightGBM machine learning with HR domain rules — flagging high-risk employees weeks before they resign.

Most HR tools notice turnover after the resignation letter arrives. This app uses 10 behavioral and operational features — including overtime accumulation, shift stress, and seasonal churn patterns — to predict who will quit next and trigger intervention before it's too late.

The Model

LightGBM + Rule-Based Ensemble

Two prediction paths run in parallel — the ML model for employees with sufficient data history, rule-based prediction as a robust fallback. The ensemble prevents overconfidence in either approach.

Primary Model
LightGBM Gradient Boosting

Trained on 300-employee synthetic histories (24 weeks each) with realistic behavioral patterns — underworked vs. overworked vs. balanced archetypes, recent deterioration patterns, seasonal variation. Returns a probability between 0 and 1. Model is retrained on each simulation run to reflect the current team's characteristics.

Fallback Model
HR Domain Rule-Based Prediction

Encodes 7+ years of retail management experience as explicit weights: shortage ratio × 0.8, surplus ratio × 1.0, mismatch rate × 1.5, seasonal factor × multiplier. Adjustable via custom weight sliders. Acts as both a fallback and an interpretability layer — managers can see exactly why the model gave a score.

10 Predictive Features

What drives churn prediction

7 original features + 3 new overtime features added by integrating Shift Optimizer stress analysis.

F1
avg_shortage_ratio
Average ratio of scheduled hours below requested hours. Underworked employees disengage.
F2
avg_surplus_ratio
Average ratio of hours above requested. Chronic overwork drives burnout and exit.
F3
mismatch_rate
Rate of weeks where shift preferences were not honored. FMI data: mismatch is a top-3 exit reason.
F4
trend_shortage
Recent 4-week shortage ratio minus past 20-week average. Rising trend = accelerating disengagement.
F5
trend_surplus
Recent vs. historical surplus ratio. Sudden overwork increase = burnout signal.
F6
trend_mismatch
Recent vs. historical mismatch rate. Worsening shift alignment = rising exit intent.
F7
seasonal_churn_factor
Employee-type × month coefficient. Students: 2.0× in May (graduation). Part-time housewives: 1.8× in July. Retail calendar-aware.
F8
overtime_ratio NEW
Overtime hours / total scheduled hours. FMI 2023: OT ratio >10% = 1.4× churn risk. Sourced from SO stress analysis.
F9
overtime_stress_score NEW
Composite: OT ratio × 1.5 + surplus ratio × 0.8 + low engagement amplifier. 0–1 scale. Updated from SO's real schedule if available.
F10
chronic_ot_flag NEW
Binary: 1 if OT hours ≥ 20/month AND surplus_ratio > 5%. Chronic overwork doubles exit probability within 6 months (FMI).
# Rule-based prediction (interpretable layer): chronic = (shortage_ratio × 0.8) + (surplus_ratio × 1.0) + (mismatch_rate × 1.5) acute = (trend_shortage × 0.8 + trend_surplus × 1.0 + trend_mismatch × 1.5) × 2.0 ot_risk = (overtime_ratio × 0.6) + (ot_stress × 0.3) + (chronic_ot × 0.1) # Role multiplier: MG/ASM × 1.3, SL × 1.1 score = 0.02 + (chronic + acute + ot_risk) × seasonal_factor

Overtime Integration

How Shift Optimizer feeds Retention

The three overtime features (F8–F10) are calculated from two sources — the employee CSV baseline, and the live Shift Optimizer stress result when available.

  1. SO runs and calculates shift stress

    calc_shift_stress() produces per-employee shortage_stress, preference_stress, and overtime_risk from the actual optimized schedule. This is stored in session_state['shift_stress_result'].

  2. RO reads the stress result

    When get_retention_features() runs, it first calculates baseline OT metrics from the employee CSV (overtime_hours_last_month, engagement_score). Then it checks if a shift_stress_result exists.

  3. Live stress overwrites baseline where available

    If the SO stress result contains a non-zero overtime_risk for an employee, it overwrites the baseline overtime_stress_score with the real value. This means RO reflects the actual schedule's impact — not just historical estimates.

  4. 10-feature vector feeds LightGBM

    The complete feature vector (F1–F10) is passed to the trained LightGBM model, returning a churn probability per employee. The rule-based model runs in parallel as both validation and fallback.

Outputs

What the dashboard shows

① Individual Churn Probability

Per-employee churn probability (0–1) ranked from highest to lowest. Color-coded: 🔴 ≥ 0.5 (high risk), 🟡 0.3–0.5 (medium), 🟢 <0.3 (low). Keyman flag (⭐) applied where replaceability is low.

② Cost vs. Risk Tradeoff

Stacked bar chart: Hiring Cost + Turnover Cost + Uncovered Hours Penalty + Headcount Risk, for each possible number of new hires (0–N). Blue vertical line = AI-optimal hiring count. Red dotted = currently selected.

③ Custom Weight Sliders

Adjust shortage, surplus, mismatch weights and trend multiplier in real time. The rule-based score recalculates instantly — showing how different management philosophies change the risk ranking.

④ Talent Vulnerability Matrix (from WP)

Replaceability × Retention probability scatter plot. Upper-left = Danger Zone (low replaceability + low retention). Shows which employees are both irreplaceable and likely to leave — the highest-priority intervention targets.

⑤ → HR Manager Dashboard: AI Prescriptive Actions

High-churn + high-stress employees automatically generate 1-on-1 recommendations with specific causal factors (mismatch, OT overload, understaffing burden). Actionable at the individual level.

Why It Matters

What this demonstrates

🧠
ML for HR, Not Just Finance

LightGBM is typically applied to fraud detection or credit scoring. Applying gradient boosting to churn prediction with behavioral HR features demonstrates breadth of ML application thinking.

🔗
Real-Time Feature Engineering

F8–F10 are calculated from SO's live schedule output — not static CSV values. This cross-app feature injection demonstrates pipeline thinking: each app's output becomes another app's input.

🏢
Interpretable + Accurate

The rule-based fallback model makes every prediction explainable: "your score is 0.62 because surplus_ratio × 1.0 + mismatch × 1.5 = chronic risk of 0.48." This is what a manager can actually act on.

📊
FMI 2023 Benchmarks Embedded

Seasonal churn coefficients, overtime risk thresholds, and turnover cost estimates are sourced from FMI 2023 Food Retail industry data — not invented. The model is calibrated against real industry baselines.
